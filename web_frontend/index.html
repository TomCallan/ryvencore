Execution Modes Comparison

Data Flow (Naive)

Mechanism: Data is propagated forward immediately on any output value change.

Use Cases: Simple, linear data processing pipelines.

Data Flow (Optimized)

Mechanism: Compiler runs topological sorting so each node executes at most once per sweep.

Use Cases: Complex data flows with split-merge paths where redundant calculations must be avoided.

Force Trigger

Mechanism: Prevents nodes from automatically propagating updates downstream when inputs change.

Use Cases: Staging changes or controlling propagation checkpoints without continuous recalculation.

Run Compiled

Mechanism: The visual graph is compiled into a single optimized Python script in-process. All dynamic events/session wrappers are bypassed.

Use Cases: Maximum performance backtesting, live socket feeds, and production deployments.
