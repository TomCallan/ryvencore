Execution Modes Comparison

Data Flow (Naive)

Mechanism: Data is propagated forward immediately on any output value change.

Use Cases: Simple, linear data processing pipelines. Fast to setup with no graph compilation overhead.

Data Flow (Optimized)

Mechanism: Compiler runs topological sorting so that each node is executed at most once per update sweep.

Use Cases: Complex data flows with split-merge paths ("diamonds") where redundant calculations must be avoided.

Force Trigger

Mechanism: Prevents nodes from automatically propagating updates downstream when inputs change. Output updates locally, but is only pushed downstream when the flow is manually run or triggered by a button node.

Use Cases: Staging changes or controlling propagation checkpoints without continuous automatic recalculation.
